# Forge 视频片段抽帧决策区优化计划

日期：2026-06-07

## 背景

真实 UI 中，用户导入视频后会在左侧看到“抽取帧、开始、结束、每隔、循环开始、循环结束”等控件。功能是完整的，但信息层级像调试面板：用户需要先理解秒、帧、循环范围等参数，才能判断下一步。

本次目标是把左侧从“参数输入区”改成“抽帧决策区”：先帮助用户判断抽哪一段、预计多少帧，再把高级参数收起。

## 产品设计原则

1. 默认工作流保持快速：导入视频后可直接抽取默认前 1 秒。
2. 解释默认行为：如果视频 10 秒但默认只抽 1 秒，UI 必须明确说明。
3. 主动作携带结果预期：按钮不只写“抽取帧”，而是写“抽取约 24 帧”。
4. 高级参数延后：循环开始/结束在未抽帧前不应和时间范围同级。
5. 保持现有暗色桌面工作台样式，不引入新的视觉语言。

## 实施任务

1. 将标题改为“视频片段”，副信息显示总时长、FPS、预计总帧数。
2. 增加范围预设：前 1 秒、全部视频、自定义。
3. 开始/结束输入显示格式化时间，避免用户误解单位。
4. 增加“预计抽取 X 帧”的结果提示。
5. 将“每隔”改为“抽帧间隔：每 X 帧”。
6. 将循环范围放入“高级设置”折叠区。
7. 主 CTA 改为动态文案：抽取约 X 帧。
8. 更新中文/英文文案和源码回归检查。

## 验收标准

1. 导入视频未抽帧时，左侧能看到视频总时长、FPS、总帧数。
2. 用户可点击“前 1 秒 / 全部视频 / 自定义”调整范围。
3. 开始/结束字段旁显示格式化时间。
4. 抽帧主按钮显示预计帧数。
5. 循环范围不再和基础抽帧参数同级。
6. `npm run test:scripts`、mac 前端构建、中文 smoke 通过。
