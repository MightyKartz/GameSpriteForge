# Forge 导出位置与工作台 UX 优化计划

日期：2026-06-07

## 背景

真实 UI 测试中，导出完成后用户仍然看到“导出资源包”作为主按钮，导出目录只在底部弱提示里出现，左侧抽帧设置和底部流程按钮也继续占据注意力。这个状态容易让用户不确定：资源包导到哪里、下一步是打开、验证，还是重新导出。

## 产品设计判断

1. 不在每次导出前强制弹出目录选择器。Forge 是本地生产工具，默认导出目录应保持连续工作流。
2. 需要在导出面板中明确显示导出位置，并提供“更改位置”。
3. 导出完成后主操作应变为“打开导出文件夹”，辅以“验证重新导入”和“重新导出”。
4. 左侧抽帧设置在已抽帧后默认收起为“重新抽取设置”，避免导出态仍像流程前半段。
5. 底部状态栏应根据导出状态切换操作，导出后不再显示检查/抽取/处理作为主要按钮。

## 实施任务

1. 导出面板增加“导出位置”卡片，显示默认导出目录或本次导出目录。
2. 导出面板增加“更改位置”按钮，复用设置页的本机目录选择能力。
3. 导出完成后，右侧主按钮切换为“打开导出文件夹”，并保留验证、重新导出。
4. 底部状态栏导出后切换为打开目录、验证、重新导出三项上下文操作。
5. 左侧导入表单在已有实时工作区后默认折叠为“更换来源”。
6. 左侧抽帧设置在已抽帧后默认折叠，命名为“重新抽取设置”。
7. 缩短窄栏导出等待验证文案，减少中文 UI 截断。
8. 更新源码级回归检查，确保上述 UX 不被后续改动回退。

## 验证标准

1. 中文 UI 可看到“导出位置”“更改位置”“打开导出文件夹”“重新导出”。
2. 已导出状态下，右侧主 CTA 是打开导出文件夹。
3. 已导出状态下，底部主要操作不再是检查 FFmpeg、抽取帧、处理质量。
4. 已有实时工作区后，左侧导入表单默认收起，并可展开更换来源。
5. 已抽帧后，左侧抽帧参数默认收起，并可展开重新抽取。
6. `npm run test:scripts`、mac 前端构建和 smoke 测试通过。
