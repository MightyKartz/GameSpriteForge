# Forge 视频抽帧主操作可达性优化计划

日期：2026-06-07

## 背景

视频导入后，左侧“视频片段”区域已经能说明范围、预计帧数和高级设置，但在真实 UI 中仍然需要滚动才能稳定看到“抽取约 24 帧”按钮。对于当前步骤来说，这个按钮是主动作，不应被摘要、参数或高级设置挤出可见区。

## 产品设计判断

1. “抽取约 X 帧”是当前步骤的主 CTA，必须始终可见。
2. 左侧运行摘要在“已选择来源、尚未抽帧”状态下信息过多，应压缩为一行。
3. 抽帧间隔和高级设置是参数，不应抢占主 CTA 的位置。
4. 保持桌面工作台紧凑风格，不引入弹窗或新的导航层级。

## 实施任务

1. 将抽帧主按钮移动到 `VideoSegmentPanel` 底部 sticky 动作条。
2. sticky 动作条增加副文本：当前时间范围和抽帧间隔。
3. 删除滚动内容中的独立主按钮，避免重复。
4. 给 `LiveWorkflowSummary` 增加 compact 模式。
5. 在帧步骤、来源已选择但尚未抽帧时，左侧摘要显示为单行状态。
6. 更新 i18n、CSS 和源码级回归检查。

## 验收标准

1. 在视频已导入、尚未抽帧时，不滚动左侧也能看到“抽取约 X 帧”。
2. 主 CTA 下方能看到时间范围和“每 X 帧”。
3. 运行摘要不再占用完整卡片高度。
4. `npm run test:scripts`、mac 构建、中文 smoke 通过。
5. 重新安装到 `/Applications/Game Sprite Forge.app` 并通过签名校验。
