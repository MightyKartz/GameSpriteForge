## Design Context

### Users
Forge V1 is for solo game developers and asset makers on macOS who already have local video, PNG frames, sprite sheets, or `.gsfpack` folders and need to turn them into game-ready 2D animation assets. Their primary job is a local, deterministic pipeline: import source media, inspect frame/quality issues, export a validated pack, and re-import it to confirm the loop.

### Brand Personality
Technical, trustworthy, and workbench-like. The product should feel like a focused local desktop tool, not a cloud generator, marketplace, or marketing site.

### Aesthetic Direction
Dense macOS desktop workbench with restrained dark UI, high information density, clear status signaling, and pixel/animation-production details. Visual polish should support scanning, comparison, and repeated action rather than decorative hero styling.

### Design Principles
- Make real pipeline state obvious: every visible control should either perform a local action now or clearly say it is unavailable.
- Prefer local-first confidence: avoid AI, account, cloud, marketplace, or publishing affordances in the MVP.
- Keep the primary loop prominent: Import -> Extract -> Process/Quality -> Export -> Validate/Re-import.
- Show demo/sample data only when it is unmistakably labeled as sample data.
- Avoid decorative complexity that competes with frame inspection, quality metrics, and export readiness.

### Current UI State
As of 2026-06-05, the production UI is no longer judged by 1:1 Figma fidelity. It should be judged by truthful local workflow behavior: sample data is labeled, unavailable workflow tabs are locked, export readiness lists blockers, live workspaces show a Run Summary, and pipeline failures expose recovery actions. Do not reintroduce fake macOS traffic lights, Marketplace, Creator Publish, BYOK, online registry, cloud generation, or provider-cost UI into the MVP.
